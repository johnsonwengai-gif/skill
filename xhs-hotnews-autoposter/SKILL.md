# 小红书热点自动成稿与发布

## 你要做什么

执行一个端到端流程：**找热点 → 选Top10 → 写小红书文案 → 出配图 → 自动发布**。

> 重要限制：小红书属于强交互平台，登录、验证码、风控等可能需要用户手动接管浏览器完成；技能应在“已登录或可手动登录”的前提下运行。

## 核心工作流（严格按顺序）

### 1) 获取当天最新热点（新闻侧）

1. 使用 `search_web`（topic=`news`，language=`zh`，time=`past day`）做 **3–6 次并行搜索**，覆盖不同信息源与角度：
   - 「今日 热点 新闻 Top」
   - 「热搜 榜 单 今日」
   - 「突发 新闻 今日」
   - 「社会 热点 今日」
   - （可选）按垂类：科技/娱乐/体育/财经
2. 对每条候选热点，优先用 `get_web_page_contents` 拉取原文/详情页（至少覆盖 Top20 候选中的 10 条）。
3. 去重与归并：同一事件来自不同媒体只算一个热点。

**筛选规则（用于Top10）：**
- **时效性**：24小时内优先
- **传播性**：多家媒体/多渠道同时出现优先
- **讨论空间**：是否容易写成“观点/科普/情绪共鸣/行动建议”
- **合规性**：避免暴力血腥、低俗、隐私泄露、煽动性内容

**输出中间结果（建议保留到草稿区）：**
- Top10 热点清单：`标题 + 1句概括 + 1条来源链接`

### 2) 生成“小红书风格”文案（成套可发）

对 Top10 生成一篇「今日热点速览」图文内容：

**文案结构（必须包含）：**
- 标题：20–28字，强信息密度 + 情绪/利益点
- 开场：1–2句说明“今天10个热点，3分钟看完”
- 正文：10条分点，每条 2–4 句：
  - 发生了什么（事实）
  - 为什么重要（影响/看点）
  - 你可以怎么做（观点/建议/关注点）
- 结尾互动：让用户评论/投票（例如“你最关注哪条？1–10选一个”）
- 话题标签：10–18个，混合：大词（#热点 #今日份）+垂类（#科技 #娱乐）+事件词

**风格要求：**
- 口语化、短句、少官腔
- 尽量避免未经证实的结论；对不确定信息用“据报道/多家媒体称”
- 不要写“引流到站外/私信领取资料”等高风险话术

> 详细提示词模板见 `references/prompts.md`。

### 3) 生成小红书配图（3:4竖图）

生成 1–3 张图（建议3张更稳定）：
- 图1：封面（「今日热点Top10」+日期）
- 图2：1–5条要点
- 图3：6–10条要点 + 结尾互动

**必须遵守：**
- 用 `generate_image` 生成，aspect_ratio=`3:4`
- 文案要“可读”：留足留白、字号偏大、对比强
- 避免使用媒体logo/截图式拼贴，尽量用“信息卡片/杂志版式”

### 4) 自动发布到小红书（浏览器自动化）

1. 用 `browser_activate` 打开小红书网页端创作入口（见 `references/xhs_publish_web.md`）。
2. 检查是否已登录：
   - 若未登录/出现验证码/风控：**必须请求用户接管浏览器**完成登录。
3. 新建图文：上传图片 → 粘贴标题/正文 → 添加话题 → 选择可见范围/定时（如有） → 发布。

**发布前自检清单：**
- 标题不夸张承诺、不涉敏感
- 正文每条热点均有“来源/据报道”语气
- 话题数量合理（10–18）且相关

## 失败与降级策略

- 若新闻源分歧大：以“多家主流媒体一致报道”的条目优先，或减少到 Top5。
- 若图片文字渲染不清晰：减少文字量，改为“3张图分摊内容”。
- 若自动发布被阻断：退化为“生成可复制的发布包”（文案+图片路径+发布步骤），由用户手动发布。
