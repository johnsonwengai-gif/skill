# 小红书网页端发布参考

## 推荐入口（可能变化）

- 主页：https://www.xiaohongshu.com/
- 创作相关入口：通常在登录后右上角“发布/创作”按钮或个人中心

> 小红书页面经常A/B测试。自动化应以“识别页面文本 + 多路径兜底”为主，而不是依赖固定CSS选择器。

## 自动发布流程（建议实现策略）

1. 打开主页，检测登录态：
   - 若出现“登录/注册”按钮或弹窗 → 需要用户接管完成登录
2. 进入发布页：
   - 优先点击含“发布”“创作”“发布笔记”等文本的按钮
3. 选择“图文/笔记”：
4. 上传图片（1–9张均可）：
5. 填写标题与正文：
   - 标题通常是单行输入
   - 正文是多行 textarea / 富文本
6. 添加话题：
   - 若支持“#话题”自动联想，逐个输入并回车确认
7. 发布或定时：
   - 若出现风控/验证码/二次确认 → 请求用户接管

## 常见阻断点

- 滑块/验证码/短信验证
- 频繁发布触发风控
- 上传失败（网络、格式、尺寸）

## 建议的降级输出（无法自动发时）

- 生成“发布包”：
  - 标题（可复制）
  - 正文（可复制）
  - 话题列表（可复制）
  - 图片文件路径（或打包成zip）
  - 手动发布步骤（3–5步）
